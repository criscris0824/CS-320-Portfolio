# CS 320 Portfolio Reflection

## How can I ensure that my code, program, or software is functional and secure?

I make sure my code works by testing every feature and making sure it does what it is supposed to do. In this course, I used JUnit tests to check different situations, including normal inputs and invalid inputs. Writing tests helped me find mistakes early before they became bigger problems. I also made sure to validate user input so bad data could not be added to the program. Keeping the code organized and testing often helped make the software more reliable and secure.

## How do I interpret user needs and incorporate them into a program?

I start by reading the project requirements carefully and breaking them down into smaller tasks. Then I think about what the user wants the program to do and how they would use it. While working on the Contact Service project, I made sure every requirement was included, like limiting the length of names and making sure IDs could not be changed. I also created tests to prove that the program met those requirements.

## How do I approach designing software?

I like to break a project into smaller pieces instead of trying to build everything at once. First, I plan the classes and what each one is responsible for. Then I write the code step by step and test each part before moving on. If something does not work, it is easier to fix because I know which section caused the problem. This course taught me that planning, testing, and making small improvements along the way leads to better software.
